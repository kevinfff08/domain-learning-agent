# Material Integrator

Exports all generated learning materials into polished, interconnected
formats: an Obsidian vault with Map of Content, glossary, and wikilinked
concept notes; a combined PDF via pypandoc; and packaged HTML. Serves as
the final output stage of the learning pipeline.

## Quick Reference

| Item | Value |
|---|---|
| Layer | 4 - Output & Tracking |
| CLI command | `newlearner export --formats "obsidian,anki,html,pdf"` |
| Python module | `src/skills/material_integrator.py` |
| Key class | `MaterialIntegrator` |
| Input | All content from `data/content/`, knowledge graph, progress data |
| Output | `output/obsidian/`, `output/export.pdf`, `output/html/` |
| Data models | Uses types from `src/models/content.py` and `src/models/knowledge_graph.py` |

## Step-by-Step Instructions

### Export to Obsidian Vault

1. Call `MaterialIntegrator.export_obsidian(graph, content_dir, output_dir)`.
2. Generates an Obsidian-compatible vault structure:
   ```
   output/obsidian/
     _MOC.md               # Map of Content - master index
     _Glossary.md           # All key terms with definitions
     concepts/
       {concept_name}.md    # One note per concept
   ```

3. **_MOC.md (Map of Content)**:
   - Groups concepts by category (Prerequisites, Core, Methods, Advanced, Practical).
   - Each concept is a wikilink: `[[concept_name]]`.
   - Shows completion status checkboxes.
   - Includes a visual progress summary.

4. **_Glossary.md**:
   - Alphabetical listing of all key terms across all concepts.
   - Each term links to its source concept note.

5. **Concept notes** (`concepts/{concept_name}.md`):
   - YAML frontmatter: concept_id, category, difficulty, status, tags,
     related concepts, key papers.
   - Three-layer content structure matching Deep Researcher output:
     Intuition, Mechanism (with LaTeX equations), Practice.
   - Wikilinks to prerequisite and related concepts: `[[other_concept]]`.
   - Embedded exercises and resource links.
   - Footer with source paper references.

### Export to PDF

1. Call `MaterialIntegrator.export_pdf(graph, content_dir, output_path)`.
   - Requires `pypandoc` (install via `pip install domain-learning-agent[pdf]`).
   - Combines all concept Markdown files in topological order.
   - Adds a table of contents, chapter headings, and page numbers.
   - LaTeX equations rendered natively by pandoc.
   - Output: single PDF at `output/export.pdf`.

### Export with Format Selection

1. CLI: `newlearner export --formats "obsidian,anki,html,pdf"`.
   - Comma-separated list of formats.
   - `obsidian`: vault export as described above.
   - `anki`: delegates to SpacedRepetitionManager.export_anki().
   - `html`: static HTML site with navigation (uses Jinja2 templates).
   - `pdf`: combined PDF as described above.
2. All formats can be generated in a single command.

## Key Implementation Details

- Obsidian wikilinks are generated by matching concept names in the
  knowledge graph; only existing nodes become links.
- YAML frontmatter in concept notes follows Obsidian's property format
  for compatibility with Dataview plugin queries.
- PDF export uses pypandoc with `--toc --number-sections` flags.
- HTML export uses templates from `templates/html_export/`.
- Jinja2 templates: `templates/obsidian_note.md.j2`,
  `templates/moc.md.j2`, `templates/glossary.md.j2`.
- Export is idempotent: re-running overwrites previous output cleanly.

## Anti-Patterns

- **Do not export before content generation is complete.** Exporting a
  partially generated vault produces broken wikilinks and empty concept
  notes. Wait until all concepts in the current learning frontier have
  been synthesized and verified.

- **Do not edit exported Obsidian notes and expect them to persist.**
  Re-running `newlearner export` overwrites the output directory. If you
  want to add personal annotations, use a separate Obsidian vault folder
  that is not in the export path.

- **Do not use PDF export for iterative study.** PDF is a snapshot format
  suited for archival or printing. For active study with evolving content,
  use the Obsidian vault which can be re-exported incrementally.
