# Textbook Planner

Searches academic APIs (OpenAlex, arXiv) for survey papers and key publications,
then uses LLM to generate a structured textbook outline with 15-30 chapters.
Replaces the former Domain Mapper / knowledge graph approach.

## Quick Reference

| Item | Value |
|---|---|
| Layer | 1 - Assessment & Planning |
| CLI command | `newlearner outline <course_id>` |
| Python module | `src/skills/textbook_planner.py` |
| Key class | `TextbookPlanner` |
| Input | `AssessmentProfile` from Pre-Assessor |
| Output | `data/courses/{course_id}/textbook.json` |
| Data models | `src/models/textbook.py` (Textbook, Chapter, ChapterStatus, PaperReference) |
| External APIs | OpenAlexClient, ArxivClient |

## Step-by-Step Instructions

### Generate a Textbook Outline

1. Ensure the course exists and has an `assessment_profile.json`.
2. Call `TextbookPlanner.generate_outline(profile, on_progress)`.
   - Searches OpenAlex and arXiv for survey/review papers on the field.
   - Passes paper summaries + learner profile to LLM.
   - LLM returns a list of chapters, each with:
     `chapter_number`, `title`, `description`, `difficulty`, `estimated_hours`,
     `key_topics`, `tags`.
   - Builds a `Textbook` object with ordered `Chapter` entries.
3. Textbook is saved to `data/courses/{course_id}/textbook.json`.

### How Chapter Generation Works (downstream)

After the outline is built, chapter content is generated by the 5-step pipeline
in the orchestrator:
1. **Deep Researcher** — synthesizes three-layer content (Intuition/Mechanism/Practice)
2. **Accuracy Verifier** — cross-checks claims against sources
3. **Resource Curator** — finds supplementary resources
4. **Quiz Engine** — generates chapter quiz
5. **Practice Generator** — creates exercises

Generation supports both batch (all chapters) and on-demand (single chapter).

## Key Implementation Details

- Paper search priority: OpenAlex (free, high quality) > arXiv (fallback).
- LLM prompt includes learner background (math/programming/domain levels),
  learning goal, and learning style for personalized outline.
- Chapter difficulty ranges from 1-5, progressing from foundational to advanced.
- `key_topics` per chapter guides downstream content generation.
- Survey papers are stored as `PaperReference` in `textbook.survey_papers`.

## Anti-Patterns

- **Do not generate an outline without an assessment profile.** Chapter depth
  and difficulty progression depend on the learner's background.

- **Do not expect stable chapter IDs across regeneration.** If you regenerate
  an outline, chapter IDs will change. Existing content for old chapters
  will be orphaned.
